## COMPONENT DETAILS

### useJobs.js hook
Fetch ALL jobs from Supabase on mount:
const { data, error } = await supabase
  .from('job_listings')
  .select('*')
  .order('date_posted', { ascending: false })

Store in state. All filtering/searching happens CLIENT-SIDE on the fetched array (do not use Supabase filters — fetch everything once).

Show loading spinner while fetching.

### ROLE CATEGORY MAPPING (in jobData.js)
Write a function getRoleCategory(job_title) that returns one of these categories based on keyword matching in the title (case-insensitive):

- "SDR" → title includes: sdr, sales development representative, sales development specialist
- "BDR" → title includes: bdr, business development representative
- "Account Executive" → title includes: account executive
- "Business Development" → title includes: business development manager, business development executive, business development associate, bdm, bde, bd manager
- "Account Manager" → title includes: account manager, key account
- "Lead Generation" → title includes: lead generation
- "Marketing" → title includes: marketing, campaign, crm specialist, performance marketing, demand generation, growth marketing, marketing automation, marketing analyst
- "Customer Success" → title includes: customer success
- "Sales Manager" → title includes: sales manager, area sales, territory sales, sales executive, sales consultant, inside sales
- "RevOps" → title includes: revops, revenue operations, sales operations
- "Other" → fallback for everything else

---

### WORK MODE DETECTION (in jobData.js)
Write a function getWorkMode(job_title, location) that returns "Remote", "Hybrid", or "Onsite":
- "Remote" if title or location contains "remote" or "wfh" or "work from home"
- "Hybrid" if title contains "hybrid"
- "Onsite" otherwise

---

### DATE FORMATTING (in jobData.js)
Write a function getRelativeDate(date_posted) that returns:
- "Today" if same calendar day as now
- "Yesterday"
- "X days ago" for up to 29 days
- "X months ago" for 30+ days

## IMPORTANT IMPLEMENTATION NOTES
1. Install dependencies: npm install @supabase/supabase-js
2. Add Tailwind properly: npx tailwindcss init -p
3. Add Inter font via Google Fonts import in index.css
4. All filtering is client-side — fetch once, filter in memory
5. The seniority_level field can be empty string for some records — handle gracefully as "Not Specified"
6. Some apply_link values contain encoded URLs with tracking params — use them as-is
7. location field contains values like "India", "Bengaluru, Karnataka, India", "Greater Bengaluru Area" — location filter should be a simple case-insensitive includes() check
8. Do NOT use any mock/static data — all data must come from Supabase
9. Make sure Tailwind classes compile correctly — use standard utility classes only

Start by creating the Supabase client, then the data utility functions, then build components bottom-up: JobCard → JobList → FilterSidebar → Hero → Navbar → App.
